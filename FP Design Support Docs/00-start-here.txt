================================================================================
00 FLASHPOINT AUTHORING — START HERE
================================================================================
One-page map of the Lore-Packs flashpoint design doc set for BattleTech Extended
Tactics (BEXT). Read this first, then open the numbered guides as you need them.
Repository: https://github.com/mattacma-cloud/Lore-Packs

The numbering order was set up to assist an AI agent in approaching the information
therefore the reading order below is somewhat different for a human flashpoint
designer seeking to work their way through the documents.

================================================================================
RECOMMENDED ORDER
================================================================================

  1. 03-story-scope-assistant.txt
     Story scale, characters, and beats before any JSON.

  2. 04-agent-design-template.txt
     Fill worksheets (Parts 1–8). Hand to an AI agent or use yourself as a spec.
     Can be read alongside 01, 02, 05 and 06 for context/details.

  2b. 07-missions-maps-objectives-guids.txt (before or while filling 04 Part 4)
     Choose mission shape, MapID, encounter layer, and GUID rules; use the map×shape
     matrix. Read 01 §3.3 when you need exact contract field names.

  3. As needed (any order after 04 is started):
     04b-conversation-design-template.txt — Leopard / dropship conversations (ConverseTek → .bytes)
     05  — Opposition paint, heraldry, map icons (Part I)
     06  — Calendar offer windows, ForcedTimelineEvent, sim-day timeouts
     07  — Tactical missions (if not already read for Part 4)

  4. 01-creation-reference.txt
     Field definitions, examples, and checklists while authoring or debugging. This
     acts as the core reference and training document for an agent assisting a 
     flashpoint designer

  5. 02-file-relationships-pathway.txt
     “Which file points to which” tree when wiring mod.json and folders.

================================================================================
WHAT EACH FILE IS FOR
================================================================================

  00  This page — where to start and how the set fits together.

  01  Canonical reference (JSON fields, GUIDs, chapters 1–8, appendix).
      Look up a specific def or behaviour; do not read cover-to-cover first.

  02  Relationship pathway (mod.json → FlashpointDef → milestones → contracts
      → lance → mech/pilot; parallel branches for events, heraldry, timeline).

  03  Story scope before modding (idea, scale, cast, pacing, lore).

  04  Designer/agent template — prompts and tables that become mod files.
      Part 1.7 / §2.0 = calendar & timeouts; Part 5.4 / 1.6 = icons;
      Part 6 = conversations (detail in 04b); Part 7.2 = calendar sim events.

  04b Conversation-only pipeline (worksheets → ConverseTek JSON → .bytes;
      milestone hex ids). Use with 01 §3.4 and 04 Part 6.

  05  Opposition units: HeraldryDef, customHeraldryDefId, regiment paint (A–C);
      tactical map markers and battle emblems (Part I).

  06  Date-specific flashpoints: timeline_* RequirementTags / ExclusionTags,
      ForcedTimelineEvent + SimGameEventDef, AvailableTimeout (sim days).
      No startDate/endDate on the poster JSON itself.

  07  Tactical missions: MapID, encounter layers, mission shapes (battle, escort,
      fire mission, three-way), objectives, GUID pairing, compatibility matrix.

QUICK “WHICH DOC?”
================================================================================

  I need to know what a field does           → 01 (chapter for that folder)
  I need to see how files link               → 02
  I am planning the story                    → 03
  I am writing the FP spec for an agent      → 04 (+ 04b / 05 / 06 / 07 worksheets)
  Which map and mission type go together     → 07 (Parts D–F, H / M.5 matrix)
  Mission will not load / objectives stuck   → 07 Part C, K.5; then 01 §3.3
  Fire mission / target acquisition          → 07 Part F.2
  Enemy colours or regiment paint            → 05 (Parts A–C)
  Wrong icon on map or wrong turn badge      → 05 Part I
  FP only offered between two calendar dates → 06 Part C (+ 04 Part 1.7)
  News pop-up or tag on a specific date      → 06 Part D (+ 04 Part 7.2)
  Offer expires after N in-game days         → 06 Part C.5 (+ 04 §2.0)
  Leopard briefing dialogue                  → 04b (+ 04 Part 6)
